# Recording and upload checklist

## Before recording

- [ ] Browser viewport is 1920x1080.
- [ ] Browser zoom is 125-150% only when it improves card readability.
- [ ] Bookmarks bar, profile icon, notifications, and unrelated tabs are hidden.
- [ ] No credential, token, local absolute path, or terminal history is visible.
- [ ] Public Code Profile, Runtime Review, Fast GCP Review, verified Fast Review,
      and More Data Rescore all return HTTP 200.
- [ ] Narration says the full name `Ops Evidence Synthesis` before `OES`.
- [ ] Narration defines stream_v3 before showing its evidence.
- [ ] Google Cloud wording uses `Gemini Enterprise Agent Platform API`.

## During editing

- [ ] Final duration is 2:50-3:05.
- [ ] Japanese subtitles are present throughout.
- [ ] The first 18 seconds identify both product and target system.
- [ ] Only one Runtime Review target is explained.
- [ ] `0.74` is called review priority, never probability.
- [ ] `0 Primary` is framed as safe restraint, not a failed analysis.
- [ ] Source Profile is called interpretation context, not runtime evidence.
- [ ] The switch to amazon-notify is explicitly introduced as genericity proof.
- [ ] Fast GCP uses the single-provider 2,000-row path.
- [ ] More Data Rescore is described as a fixed child-bundle re-score, not a new
      live model call.
- [ ] Agent Runtime deployment is not claimed.
- [ ] Raw log upload is stated as zero / not_uploaded.

## Before upload

- [ ] Watch once with sound off: the subtitles still tell the story.
- [ ] Watch once at 1.5x: the key numbers and target names remain readable.
- [ ] Verify every URL in the video description in an incognito window.
- [ ] Use an unlisted YouTube URL until the final version is approved.
- [ ] Thumbnail contains `AIの同意 ≠ 原因` and the full product name.
- [ ] ProtoPedia receives the architecture image and final video URL.
- [ ] The final submission form receives GitHub, deployed project, and
      ProtoPedia URLs.
